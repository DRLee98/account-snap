## 목적

[[OCR-002]] 파싱 결과를 사용자에게 표시하고, 복사·저장(즐겨찾기)·재촬영 액션 제공.

## UI 구성

```
┌──────────────────────────┐
│ [←]    계좌 정보  [재촬영]│
│                          │
│  🏦 국민은행              │  ← 은행 (편집 가능)
│  ┌────────────────────┐  │
│  │ 110-1234-5678-9012 │  │  ← 계좌번호 (편집 가능)
│  └────────────────────┘  │
│  👤 김장보 (예금주)        │  ← OCR로 잡혔으면 표시
│                          │
│  [별칭 추가 +]            │  ← 사용자 라벨
│                          │
│  ┌──────────────────┐    │
│  │  📋 계좌번호 복사 │    │  ← 메인 액션
│  └──────────────────┘    │
│  ┌──────────────────┐    │
│  │  ⭐ 즐겨찾기 추가 │    │

│  ──── OCR 후보 ────       │  ← candidates 있을 때만
│  • 110-1111-... (신한)   │
```

## 기능

### 1. 표시 + 편집
- 은행명, 계좌번호, 예금주, 별칭 모두 인라인 편집 가능
- OCR 신뢰도 낮으면 (< 0.7) 필드 옆에 ⚠️ 표시 + "확인해주세요" 힌트

### 2. 복사
- 탭 시 계좌번호만 (정규화된 숫자) 클립보드 복사 → 토스트 "복사됨"
- 동시에 `lastUsedAt` 업데이트 ([[DAT-001]])

### 3. 즐겨찾기 추가
- 토글 시 `isFavorite` true + 저장
- 별칭 입력 모달 (선택 사항)

### 4. 자동 저장
- 화면 진입 시점에 일단 history로 저장 (isFavorite=false)
- 사용자가 즐겨찾기 토글하면 isFavorite만 갱신

### 5. 후보 선택
- [[OCR-002]]의 `candidates`가 여러 개면 하단 리스트로 표시
- 탭 시 메인 영역으로 스왑

## 위젯 갱신 트리거

- 복사 또는 즐겨찾기 추가 시 → MMKV `accounts:lastUsedId` 갱신 → iOS `WidgetCenter.reloadAllTimelines()` / Android `AppWidgetManager.notifyAppWidgetViewDataChanged()` 호출 ([[WGT-001]]의 브릿지 사용)
